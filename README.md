## 1. Общие описание
Необходимо разработать программный продукт для планирования встреч сотрудников внутри компании. Система состоит из клиентского мобильного приложения и серверной части.

## 2. Клиентская часть (Android-приложение)
Клиентом является **Android-приложение**, которое должно быть реализовано на **Kotlin** или **Java** с использованием **адаптивной верстки**.

Приложение должно предоставлять интуитивно понятный интерфейс, в котором реализован следующий функционал:

*   **Регистрация и авторизация** сотрудника.
*   **Настройка личного профиля**.
*   **Создание встречи** с возможностью приглашения других сотрудников. Приглашенные сотрудники должны иметь возможность дать бинарный ответ (**принял** / **не принял**).
*   **Просмотр списка активных приглашений** на встречи с возможностью дать ответ о возможности посещения.
*   **Просмотр собственного расписания встреч** в разрезе дня, недели и месяца.

**Важное условие:** временные слоты для встреч разбиты строго по часам. Например, можно создать встречу на 9:00–10:00, но создание встречи на 9:10, 9:05 и т.д. — невозможно.

## 3. Серверная часть
Сервер представляет собой **микросервис на Spring Boot**, работающий с СУБД **PostgreSQL** или **H2**.

Серверное приложение должно:
*   Предоставлять API, необходимый для полноценной работы мобильного приложения.
*   Осуществлять все основные операции с данными в базе через **Spring Data JPA**.
*   Использовать **Liquibase** для создания схемы базы данных и ее предзаполнения.


## User Stories

### US-1. Регистрация и авторизация
Как сотрудник, я хочу зарегистрироваться и авторизоваться, чтобы получить доступ к функционалу приложения.

### US-2. Управление профилем
Как сотрудник, я хочу просматривать и редактировать свой профиль, чтобы мои данные были актуальны.

### US-3. Создание встречи
Как сотрудник, я хочу создавать встречи с приглашением других сотрудников, чтобы планировать рабочие мероприятия.

### US-4. Управление приглашениями
Как сотрудник, я хочу видеть приглашения на встречи и отвечать на них (принять / отклонить).

### US-5. Просмотр расписания
Как сотрудник, я хочу просматривать своё расписание по дням, неделям и месяцам.


## Use Cases

### US-1: Регистрация и авторизация

#### UC-1.1 Регистрация
**Предусловия:** пользователь не авторизован  
**Сценарий:**
1. Ввод email, пароля, ФИО
2. Отправка формы
3. Создание пользователя в системе  
**Постусловия:** пользователь зарегистрирован

#### UC-1.2 Авторизация
**Предусловия:** пользователь зарегистрирован  
**Сценарий:**
1. Ввод email и пароля
2. Проверка данных
3. Получение токена  
**Постусловия:** пользователь авторизован

---

### US-2: Профиль пользователя
**Предусловия:** пользователь авторизован  
**Сценарий:**
1. Просмотр профиля
2. Редактирование данных
3. Сохранение изменений  
**Постусловия:** профиль обновлён

---

### US-3: Создание встречи
**Предусловия:** пользователь авторизован  
**Сценарий:**
1. Выбор даты и часового слота (строго по часам)
2. Добавление описания и участников
3. Создание встречи  
**Постусловия:** встреча создана, приглашения отправлены

---

### US-4: Приглашения
**Предусловия:** пользователь авторизован  
**Сценарий:**
1. Просмотр списка приглашений
2. Принятие или отклонение приглашения  
**Постусловия:** статус приглашения обновлён

---

### US-5: Расписание
**Предусловия:** пользователь авторизован  
**Сценарий:**
1. Выбор режима (день / неделя / месяц)
2. Просмотр встреч  
**Постусловия:** отображено актуальное расписание


## Задачи backend-разработчика

- Настроить Spring Boot проект (JPA, Security, Liquibase)
- Создать модель данных: User, Profile, Meeting, Invitation
- Реализовать регистрацию и авторизацию (JWT)
- Реализовать API профиля
- Реализовать создание встреч с проверкой часовых слотов
- Реализовать API приглашений
- Реализовать API расписания
- Добавить Liquibase и тестовые данные


## Задачи frontend-разработчика

- Создать Android-приложение (Kotlin/Java)
- Реализовать экраны регистрации и входа
- Реализовать экран профиля
- Реализовать создание встречи
- Реализовать экран приглашений
- Реализовать календарь (день / неделя / месяц)
- Интеграция с backend API
